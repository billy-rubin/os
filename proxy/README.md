# Multithreaded Caching HTTP Proxy

Многопоточный кэширующий HTTP-прокси сервер, реализующий поддержку протокола HTTP 1.0 (GET запросы). Проект разработан в рамках курса "Операционные системы".

## Возможности
* **HTTP 1.0**: Поддержка GET запросов.
* **Кэширование**: Сохранение ответов сервера в памяти. Если несколько клиентов запрашивают один и тот же ресурс, запрос к целевому серверу выполняется только один раз, а данные раздаются всем клиентам параллельно (writer/readers synchronization).
* **LRU Cache**: При переполнении кэша удаляются старые, давно не используемые записи.
* **Пул потоков**: Обработка соединений происходит в фиксированном пуле потоков, что ограничивает нагрузку на систему.
* **Garbage Collection**: Отдельный поток фоном очищает кэш при необходимости.

## Структура проекта

* `proxy/` - Основная логика прокси-сервера.
* `proxy/cache/` - Реализация кэша (хеш-таблица с LRU вытеснением).
* `proxy/logger/` - Подсистема логирования.
* `proxy/threadpool/` - Реализация пула потоков.

## Требования

* Linux/Unix environment
* GCC (с поддержкой C11/C17)
* Make
* Pthread library
