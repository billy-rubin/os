# Multithreaded Caching HTTP Proxy

Многопоточный кэширующий HTTP-прокси сервер, реализующий поддержку протокола HTTP 1.0 (GET запросы). Проект разработан в рамках курса "Операционные системы".

## Возможности
* [cite_start]**HTTP 1.0**: Поддержка GET запросов[cite: 336].
* **Кэширование**: Сохранение ответов сервера в памяти. [cite_start]Если несколько клиентов запрашивают один и тот же ресурс, запрос к целевому серверу выполняется только один раз, а данные раздаются всем клиентам параллельно (writer/readers synchronization)[cite: 327].
* [cite_start]**LRU Cache**: При переполнении кэша удаляются старые, давно не используемые записи[cite: 377].
* [cite_start]**Пул потоков**: Обработка соединений происходит в фиксированном пуле потоков, что ограничивает нагрузку на систему[cite: 356].
* [cite_start]**Garbage Collection**: Отдельный поток фоном очищает кэш при необходимости[cite: 378].

## Структура проекта

* `proxy/` - Основная логика прокси-сервера.
* `proxy/cache/` - Реализация кэша (хеш-таблица с LRU вытеснением).
* `proxy/logger/` - Подсистема логирования.
* `proxy/threadpool/` - Реализация пула потоков.

## Требования

* Linux/Unix environment
* GCC (с поддержкой C11/C17)
* Make
* Pthread library
